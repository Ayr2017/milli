## ****Миграция 001_initial.sql — создание схемы для системы очередей.****

**Назначение**: Создаёт две основные таблицы для работы фоновых заданий по принципам, аналогичным Laravel Queues.
**jobs**: Хранит активные задания (ожидающие, выполняющиеся, завершённые).
**failed_jobs**: Архив заданий, завершившихся с ошибкой после исчерпания всех попыток.

### Ключевые моменты:

* Статус (status): Определяет текущее состояние задания: pending, running, failed.
* Полезная нагрузка (payload): Поле типа TEXT для хранения сериализованных данных задачи (например, в JSON).
* Повторы (attempts/max_attempts): Механизм для автоматических повторных попыток выполнения.
* Планирование (scheduled_at): Поддержка отложенного запуска задания.
* Индексы: Созданы для оптимизации основных запросов: выборка задач по очереди и статусу, работа с отложенными задачами и сортировка ошибок.
* Безопасность: Использование IF NOT EXISTS гарантирует, что миграцию можно запускать многократно без ошибок.
